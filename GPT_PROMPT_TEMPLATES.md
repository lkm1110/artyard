# GPT 프롬프트 템플릿 모음

## 📋 사용 방법

1. 아래 템플릿 복사
2. `[...]` 부분을 실제 내용으로 교체
3. GPT에게 붙여넣기
4. 결과 확인 및 피드백

---

## 🎯 기본 프롬프트 (항상 먼저 사용)

```
나는 ArtYard라는 모바일 아트 마켓플레이스를 개발하고 있어.

플랫폼 개요:
- 대학생 아티스트와 구매자를 연결
- React Native (Expo) + TypeScript
- Supabase (PostgreSQL + Real-time)
- 10% 플랫폼 수수료 (판매자가 90% 수령)

자세한 내용은 아래 문서를 참고해줘:

[PLATFORM_BRIEF_FOR_GPT.md 전체 내용 붙여넣기]

이해했으면 "ArtYard 플랫폼을 이해했습니다. 어떤 문서화 작업을 도와드릴까요?" 라고 답변해줘.
```

## 📚 문서 타입별 템플릿

### 1. API 문서화

```
ArtYard의 [파일명 예: artworkService.ts] 파일에 있는 모든 함수를 API 문서로 작성해줘.

포함할 내용:
- 함수 시그니처 (TypeScript 타입 포함)
- 파라미터 설명 (타입, 필수/선택, 기본값)
- 반환값 타입 및 설명
- 에러 케이스 및 처리 방법
- 사용 예시 (코드 스니펫)
- 관련 DB 테이블
- 성능 고려사항 (있다면)

형식: Markdown, 개발자용

실제 코드:
[코드 파일 내용 붙여넣기]
```

### 2. 사용자 가이드

```
ArtYard의 [기능명] 기능에 대한 사용자 가이드를 작성해줘.

대상 사용자: [대학생 아티스트 / 구매자 / 관리자]
톤: [친절하고 격려하는 / 전문적인 / 간결한]

포함할 내용:
1. 기능 개요 (왜 사용하는가?)
2. 사전 준비사항
3. 단계별 가이드 (스크린샷 설명 포함)
4. 자주 묻는 질문 (FAQ)
5. 트러블슈팅 (일반적인 문제와 해결)
6. 팁과 베스트 프랙티스

형식: Markdown, 이미지 설명 포함

참고할 화면:
[스크린샷 또는 화면 설명 붙여넣기]
```

### 3. 아키텍처 결정 기록 (ADR)

```
ArtYard에서 [기술/라이브러리/패턴]을 선택한 이유를 ADR 형식으로 문서화해줘.

배경:
[왜 이 결정이 필요했는지]

고려한 옵션:
1. [선택한 것]
2. [대안 1]
3. [대안 2]

결정:
[최종 선택]

이유:
[선택한 이유, 장단점 비교]

결과:
[긍정적 효과]
[부정적 효과 또는 트레이드오프]

ADR 형식:
- Title: ADR-[번호]: [제목]
- Status: Accepted
- Date: 2025-01-11
- Context: ...
- Decision: ...
- Consequences: ...
```

### 4. 데이터베이스 스키마 문서

```
ArtYard의 [테이블명] 테이블을 문서화해줘.

포함할 내용:
- 테이블 목적 및 역할
- 컬럼 목록 (이름, 타입, 제약조건, 설명)
- Primary Key, Foreign Key
- 인덱스 (성능 최적화)
- RLS 정책 (누가 접근 가능한가?)
- 관련 테이블 (관계 다이어그램 설명)
- 일반적인 쿼리 예시
- 성능 고려사항

형식: Markdown, ERD 설명 포함

실제 SQL:
[CREATE TABLE 문 붙여넣기]
```

### 5. 컴포넌트 문서 (Storybook 스타일)

```
ArtYard의 [컴포넌트명] 컴포넌트를 문서화해줘.

포함할 내용:
- 컴포넌트 개요 (언제, 왜 사용하는가?)
- Props 목록 (이름, 타입, 필수/선택, 기본값, 설명)
- 사용 예시 (여러 시나리오)
- 스타일링 가이드 (테마 지원, 커스터마이징)
- 접근성 (Accessibility) 고려사항
- 성능 최적화 팁
- 관련 컴포넌트

형식: Markdown, 코드 예시 포함

실제 코드:
[컴포넌트 파일 내용 붙여넣기]
```

### 6. 트러블슈팅 가이드

```
ArtYard에서 발생한 [문제 설명] 이슈를 트러블슈팅 가이드로 작성해줘.

문제 설명:
[무엇이 잘못되었는가?]

증상:
[사용자가 보는 에러 메시지, 동작]

원인:
[왜 발생했는가?]

해결 방법:
[단계별 해결 과정]

예방 방법:
[미래에 이 문제를 방지하는 법]

관련 이슈:
[비슷한 문제들]

포함할 내용:
- 문제 재현 단계
- 에러 로그 해석
- 디버깅 팁
- 임시 해결책 (workaround)
- 영구 해결책

형식: Markdown, 코드/로그 포함
```

### 7. 온보딩 가이드 (신규 개발자용)

```
ArtYard 프로젝트에 새로 합류한 개발자를 위한 온보딩 가이드를 작성해줘.

포함할 내용:
1. 프로젝트 개요 (5분 이해)
2. 개발 환경 설정
   - 필수 도구 (Node.js, Expo, etc.)
   - 환경 변수 (.env 설정)
   - Supabase 접근 권한
3. 코드베이스 구조
   - 폴더 구조 설명
   - 주요 파일 위치
   - 네이밍 컨벤션
4. 개발 워크플로우
   - 로컬 실행 방법
   - 테스트 방법
   - 빌드 방법
5. 첫 번째 기여 가이드
   - 간단한 작업 예시
   - PR 프로세스
6. 자주 참고하는 문서 링크

대상: 신입 React Native 개발자
톤: 친절하고 격려하는

형식: Markdown, 체크리스트 포함
```

[핵심 메시지, 예: "당신의 예술로 수익을 만드세요"]

```

[2단계]

[3단계]
다음은 [섹션 2]를 작성해줘.

... (반복)
```

### 2. 예시 기반 문서화

```
ArtYard의 [기능]을 설명하는 문서를 작성해줘.

[다른 서비스의 문서 링크 또는 설명]

- [우리만의 특징 1]

```

```
